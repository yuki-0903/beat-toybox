# Tiny Toy Sprint Asset Prompts

Use this prompt sheet to generate the production image set for Tiny Toy Sprint. Start with P0, then fill P1/P2 when the core gameplay read is solid.

## Shared Style

Apply this style block to every prompt:

```txt
Toy-like miniature game asset for a rhythm runner game. Bright daytime kid's desk world, rounded plastic toy shapes, soft shadow, thick clean outline, simple readable silhouette, cute and slightly surreal, not realistic, not photo, not detailed illustration. Consistent palette: toy red #FF6B6B, toy yellow #FFD166, toy blue #4D96FF, mint #6BCB77, orange #FF9F1C, cream #FFF3D8, dark brown #3A2A1A. Centered object, transparent background, generous padding, no text, no watermark.
```

For background assets, replace `transparent background` with the background-specific note.

## Replacement Notes

Generated PNG names above are the production target names. Current MVP SVG placeholders remain as fallback. After PNGs are ready, save them at the listed paths and start the game with:

```txt
NEXT_PUBLIC_USE_PRODUCTION_ASSETS=true npm run dev -- -H 0.0.0.0 -p 3000
```

If you add assets that are not listed in `game/config/themeAssets.ts`, add a `productionFile` value and priority for the matching key.

To check remaining generated PNG work:

```txt
npm run report:assets
```

Keep every object centered with 8-12% transparent padding so Phaser scaling does not crop shadows or glow.
