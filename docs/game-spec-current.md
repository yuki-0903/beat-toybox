# BEAT RUNNER ゲーム仕様書

最終更新: 2026-05-19

## 1. ゲーム概要

`BEAT RUNNER` は、3人の小さな玩具キャラクターが音楽に合わせて演奏しながら、ミニチュア世界を進んでいくリズムランナーゲームです。

プレイヤーは3レーンに対応したキャラクターをタイミングよく操作し、奥から近づいてくるリズムキューに合わせて楽器を鳴らします。

このゲームで一番大事にする体験は、厳密な音ゲー攻略ではなく、

> 気持ちよく世界を駆け抜けること

です。

## 2. 現在の中心コンセプト

現在のBEAT RUNNERは、初期案の「障害物を避けるゲーム」から方向転換し、以下を中心体験にします。

```txt
3人の音楽隊キャラを、リズムに合わせて演奏させる
```

ゲーム画面上ではキャラクター自身が大きく移動するのではなく、背景とレーンによって「走っている世界」を見せます。奥から手前へ近づくキューに合わせてキャラクターを操作すると、キャラが楽器を奏で、BGMにSEが重なります。

### プレイ感の目標

- 音に乗れる
- 操作が演奏になる
- キャラを押すのが気持ちいい
- コンボをつなぎたくなる
- 3人の音楽隊を指揮している感じがある
- 玩具世界の中を進んでいる感じがある

## 3. ゲームループ

```txt
Start
  ↓
Music Select
  ↓
Select Level
  ↓
Game
  ↓
Result / Ranking Register
  ↓
Ranking
```

### 画面ごとの役割

| 画面 | 役割 |
|---|---|
| Start | ゲームの入口。タイトル、背景、主要ボタンを表示する。 |
| Music Select | 遊ぶ曲をカード形式で選ぶ。 |
| Select Level | 難易度を選んでゲーム開始する。 |
| Game | 3キャラをリズムキューに合わせて演奏させる。 |
| Result | スコア、コンボ、ミスなどを確認する。 |
| Ranking | 登録済みスコアを表示する。 |

## 5. Start画面仕様

### 目的

ゲームの第一印象を作る画面です。余計な情報を出しすぎず、`BEAT RUNNER` のタイトル、キャラクター、玩具世界が一目で伝わる状態にします。

### 表示内容

- フルスクリーン背景画像
- `BEAT RUNNER` タイトル
- 3人の音楽隊キャラクター
- `START!` ボタン
- ランキングボタン
- 設定ボタン
- ヘルプボタン

### 操作

| 操作 | 遷移 |
|---|---|
| START | Music Selectへ進む |
| Ranking | Rankingへ進む |
| Setting | 現状は仮表示 |
| Help | 現状は仮表示 |

### デザイン方針

- 背景は1枚絵を基本にする。
- ボタンは玩具っぽいピンク/白枠の丸みがあるデザイン。
- フォントはFredoka系の丸く太い印象を優先する。
- 文字やボタン背景が不自然に伸びないようにする。

## 6. Music Select画面仕様

### 目的

遊ぶ曲を選ぶ画面です。曲ごとの世界観がカードで分かるようにします。

### 表示内容

- 共通背景画像
- `MUSIC SELECT` 見出し
- 中央の曲カード
- 左右の曲カード
- 左右矢印
- ページドット
- 右下の設定ボタン

### 現在の仕様

- 中央カードが現在選択中の曲。
- 左右カードは前後の曲を表示する。
- 左右カードは傾けず、まっすぐ表示する。
- 戻るボタンは表示しない。
- 設定ボタンは右下に配置する。
- カード内の画像は、カードの表示領域いっぱいに見せる。

### 操作

PC:

| 入力 | 動作 |
|---|---|
| A / ← | 前の曲 |
| D / → | 次の曲 |
| Enter / Space | 曲を決定 |

SP:

| 入力 | 動作 |
|---|---|
| 左矢印ボタン | 前の曲 |
| 右矢印ボタン | 次の曲 |
| 中央カード | 曲を決定 |

## 8. Game画面仕様

### 目的

メインのプレイ画面です。3人の音楽隊キャラクターを、奥から流れてくるリズムキューに合わせて演奏させます。

### 表示内容

- フルスクリーン背景
- 3レーン
- 赤/黄/青の3キャラクター
- 奥から手前へ近づくリズムキュー
- SCORE
- COMBO
- MISS
- FEVER表示/演出
- SP用の3つの丸ボタン

### レーン

| レーン | 色 | 担当 |
|---|---|---|
| LEFT | 赤 | 赤キャラ |
| CENTER | 黄 | 黄キャラ |
| RIGHT | 青 | 青キャラ |

レーンは背景と一体化して見せます。現時点では、強い塗り面よりも線や奥行きで3レーンを認識できる状態を優先します。

### キャラクター

キャラクターは曲ごとに変えず、固定です。

| レーン | 色 | キャラクター方向 | 楽器/音 |
|---|---|---|---|
| LEFT | 赤 | 正面向き | ドラム |
| CENTER | 黄 | 正面向き | シンバル/ベル |
| RIGHT | 青 | 正面向き | キーボード/ギター |

### 操作

PC:

| 入力 | 対応キャラ |
|---|---|
| A / ← / 1 | 左キャラ |
| S / ↑ / ↓ / 2 | 中央キャラ |
| D / → / 3 | 右キャラ |

SP:

- 画面下部に3つの丸いボタンを表示する。
- 各ボタンは対応キャラクターの真下に配置する。
- ボタンは画面幅を3分割したサイズ感を基準にし、少し余白を入れる。
- ボタンを押すと、対応キャラクターが演奏モーションを行う。

### 演奏アクション

プレイヤー入力時、対応キャラクターが楽器を奏でるモーションを行います。

現在は旧仕様の名残で、コード上の一部に`jump`という名前が残っています。ただし、ゲームデザイン上の意味はジャンプではありません。

```txt
jump = 演奏アクション
```

## 9. リズムキュー仕様

### 役割

奥から手前へ近づいてくるオブジェクトです。プレイヤーはキューが判定位置に来るタイミングで、対応するキャラクターを操作します。

### 表示方針

- 正面から近づいてくる見た目にする。
- レーン幅に対して見やすいサイズにする。
- 小さすぎると反応しづらいので、スマホ縦画面を基準に調整する。
- アイテム取得ではなく、演奏タイミングを示すものとして扱う。

### 実装上の名称

現在のコードと譜面JSONでは、旧仕様の名残で`obstacle`という名前が残っています。

仕様上の意味は以下です。

```txt
obstacle = rhythm cue / performance target
```

将来的に余裕があれば、実装名を`cue`へ移行します。

## 10. 判定仕様

### 成功

キューが判定位置付近に来たタイミングで、対応キャラクターを操作すると成功です。

成功時に起こること:

- キャラクターが演奏モーションを行う
- 対応キャラのSEが鳴る
- SCOREが増える
- COMBOが増える
- `PERFECT` / `GOOD` 系の演出が出る
- 小さなパーティクルが出る

### 失敗

キューに対して入力が合わなかった場合はミスになります。

失敗時に起こること:

- MISSが増える
- COMBOがリセットされる
- キャラクターが小さく失敗リアクションをする
- `MISS` / `OOPS` 系のポップな演出が出る

注意:

- 赤い強烈なフラッシュは禁止。
- 怖い失敗演出ではなく、玩具世界らしい軽い失敗表現にする。

## 11. SCORE / COMBO / MISS

### SCORE

成功したキューに応じて加算します。将来的には判定精度や難易度で点数差をつけます。

### COMBO

連続成功数です。

COMBO増加時:

- 数字を弾ませる
- 少し拡大する
- 軽い回転やキラキラを追加する

### MISS

失敗数です。警告感は出しつつ、画面全体が重くならないようにします。

## 12. FEVER仕様

### 発動条件

一定コンボ到達でFEVER状態になります。

現在の参照値:

```ts
GAME_BALANCE.feverComboThreshold
```

### 演出方針

FEVER中は「玩具世界のテンションが上がった」状態にします。

追加/強化するもの:

- SCORE / COMBO UIの弾み
- 星/音符/丸パーティクル
- 軽いカメラ演出
- キャラクター演奏の気持ちよさ
- レーン周辺のささやかな光

禁止:

- 黒背景
- ネオン過多
- 強すぎる画面揺れ
- 視認性を壊す大量エフェクト

## 13. アイテム仕様

### 現在の状態

通常プレイ中のアイテム取得は、現時点では無効化しています。

理由:

- 現在の中心体験を「3人の演奏タイミング」に絞るため。
- アイテムとキューが混ざると、プレイヤーが何を狙うべきか曖昧になるため。

### 将来案

将来的には、演奏キューとは別のボーナス要素として復活させる可能性があります。

例:

- FEVERゲージを増やす
- 一時的にスコア倍率を上げる
- 特別な音色を追加する
- 背景演出を一段階盛り上げる

## 15. 譜面JSON仕様

譜面JSONは以下に配置します。

```txt
public/assets/charts/
```

基本構造:

```json
{
  "id": "track-id",
  "title": "Track Title",
  "bpm": 128,
  "lanes": 3,
  "duration": 80,
  "approachTime": 1.6,
  "obstacles": [
    {
      "time": 1.25,
      "lane": 0,
      "type": "toy_block",
      "pattern": "single",
      "energy": 0.7
    }
  ],
  "items": []
}
```

### lane

| 値 | レーン |
|---:|---|
| 0 | LEFT |
| 1 | CENTER |
| 2 | RIGHT |

### obstacles

現時点では`obstacles`をリズムキューとして扱います。障害物回避用のデータではありません。

### items

現時点ではゲーム内アイテムを非表示/無効化しているため、`items`は基本的に使いません。

## 16. サウンド仕様

### BGM

曲ごとにBGMファイルを持ちます。

配置場所:

```txt
public/assets/audio/
```

### キャラクターSE

3キャラそれぞれに演奏SEを割り当てます。

| キャラ | ファイル |
|---|---|
| 赤 | `se_character_red.mp3` |
| 黄 | `se_character_yellow.mp3` |
| 青 | `se_character_blue.mp3` |

### ブラウザ制約

ブラウザではユーザー操作前に音を再生できないため、BGM/SEの開始はユーザー操作後に行います。

## 20. ランキング仕様

### 現在の保存方式

ランキングはJSONファイルに保存します。

保存先:

```txt
data/rankings.json
```

API:

```txt
app/api/rankings/route.ts
```

### 注意

この方式はローカル開発/Node.jsサーバー実行向けです。GitHub Pagesのような静的ホスティングでは、実行時にJSONへ永続保存できません。

将来、プレイヤー全体でランキングを共有する場合は、外部DBやBaaSが必要です。
